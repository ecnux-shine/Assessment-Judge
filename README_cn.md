# Assessment Judge (AJ)

一个 Windows C++ 命令行工具，可以在模拟终端环境下运行 `.exe` 程序，并报告执行时间和内存占用。

## 功能特性

- **三种运行模式**：手动输入、数据测试、多数据批量测试
- **精确时间统计**：用户输入时间不计入统计
- **内存监控**：实时追踪程序内存占用
- **灵活的限制配置**：可自定义时间和内存上限
- **详细的执行报告**：显示运行状态（AC/TEL/WA/MLE）

## 编译

使用 MinGW 或 MSYS2：

```batch
g++ aj.cpp -std=c++17 -O2 -o aj.exe -lpsapi
```

使用 MSVC（Visual Studio 开发者命令提示符）：

```batch
cl /EHsc aj.cpp psapi.lib
```

## 使用方法

### 模式一：手动输入模式

运行程序并等待用户输入：

```batch
aj.exe program.exe
```

程序会模拟完整的终端环境，用户可以手动输入数据。输入时间不会计入程序运行时间。

输出示例：
```
------------------------------
Time: 152 ms
Memory: 6.3 MB
Status: AC
```

### 模式二：数据测试模式

使用 `-i` 参数传入输入数据，程序直接运行：

```batch
aj.exe program.exe -i "输入内容"
```

程序会自动将指定的输入数据发送给目标程序，不需要等待用户输入。

输出示例：
```
------------------------------
Status: TEL
```

### 模式三：多数据批量测试模式

使用 `-f` 参数指定测试数据文件：

```batch
aj.exe program.exe -f "tests.txt"
```

测试文件中的内容会按空格和换行符分割为多个令牌，按顺序分组输入给程序。

输出示例：
```
Average Time: 135ms
Average Memory: 200 MB
All Statuses: AC: 31/40      TEL: 2/40      WA: 8/40      MLE: 0/40
```

### 输出所有详细结果

在批量测试模式中添加 `-a` 参数以输出每次测试的详细结果：

```batch
aj.exe program.exe -f "tests.txt" -a
```

输出示例：
```
##########[1]##########

Time: 100ms
Memory: 5.2 MB
Status: AC

##########[2]##########

Time: 120ms
Memory: 6.1 MB
Status: AC

...
```

## 高级选项

### 覆盖单次运行的限制

使用 `-ct`（时间限制）和 `-cm`（内存限制）参数来为单次运行修改限制：

```batch
aj.exe program.exe -ct 200 -cm 200 -i "输入"
```

此命令将时间限制设为 200ms，内存限制设为 200MB。

### 修改默认限制

使用 `-c` 参数修改全局默认限制：

```batch
aj.exe -c time 200 memory 200
```

此命令将默认时间限制设为 200ms，默认内存限制设为 200MB。配置会保存在 `aj_config.txt` 中。

**注意**：`-c` 参数必须单独使用，不能与其他选项组合。

## 默认限制

- 时间限制：1000 ms（1 秒）
- 内存限制：256 MB

## 运行状态说明

| 状态 | 含义 | 说明 |
|------|------|------|
| AC | Accepted（通过） | 程序正常运行完成，内存和时间都在限制内 |
| TEL | Time Exceeded Limit（超时） | 程序运行时间超过了时间限制 |
| MLE | Memory Limit Exceeded（超内存） | 程序占用内存超过了内存限制 |
| WA | Wrong Answer（答案错误） | 程序异常退出或返回非零退出码 |

## 工作流程

### 手动模式计时逻辑

1. 程序启动时开始计时
2. 程序等待输入时**暂停**计时
3. 用户输入完成，计时**继续**
4. 程序结束后统计总时间

### 批量测试模式

1. 自动检测程序需要的输入令牌数（通过试运行）
2. 按该数量将测试数据分组
3. 逐组运行程序
4. 如果测试数据不足以填充最后一组，该组计为 WA
5. 统计所有运行的平均时间和内存

## 配置文件

程序会在当前目录创建 `aj_config.txt` 来保存默认限制：

```
time 1000
memory 256
```

可以直接编辑此文件或使用 `-c` 参数修改。

## 示例

运行一个简单的程序并等待输入：
```batch
aj.exe myprogram.exe
```

使用固定输入运行程序：
```batch
aj.exe myprogram.exe -i "5"
```

运行 10 组测试数据：
```batch
aj.exe myprogram.exe -f "test_data.txt"
```

运行测试并显示所有详细结果，时间限制为 500ms：
```batch
aj.exe myprogram.exe -f "test_data.txt" -a -ct 500
```

修改全局默认限制为时间 2000ms，内存 512MB：
```batch
aj.exe -c time 2000 memory 512
```

## 注意事项

- 目标程序必须是 Windows 可执行文件（.exe）
- 测试数据文件中的空格和换行符都被视为分隔符
- 程序终止时会清理所有资源和子进程
- 内存峰值是指程序运行期间的最大工作集大小
